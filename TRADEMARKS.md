# Trademark and Naming Policy

This policy covers the CodeIndex project name, the `cdidx` command/package name,
release badges, repository identity, and any confusingly similar names or
branding.

The copyright license in `LICENSE` does not grant trademark, service mark,
trade name, package identity, endorsement, or false-affiliation rights.

## Allowed Uses

You may use the names CodeIndex and cdidx for truthful, non-confusing references,
including:

- "compatible with CodeIndex";
- "uses cdidx if installed";
- "configuration for CodeIndex";
- "based on CodeIndex" for truthful attribution;
- links back to the official repository;
- internal-only fork names that cannot reasonably be confused with an official
  release channel.

## Not Allowed Without Written Permission

You may not:

- publish a derivative package, binary, container image, extension, service,
  or website under `CodeIndex`, `cdidx`, or a confusingly similar name;
- reuse the official repository URL, NuGet package ID, command identity,
  badges, release names, or maintainer identity for a derivative distribution;
- market a derivative as official, endorsed, approved, or maintained by Widthdom;
- use the names to sell or provide a competing CodeIndex-derived code-search,
  AI-retrieval, or MCP-index product.

Derivative distributions must use a clearly different name, package ID,
command name, repository URL, and release channel, and must keep the required
license notices.
