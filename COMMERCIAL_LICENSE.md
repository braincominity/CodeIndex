# Commercial and Competing Use Policy

CodeIndex exists to make AI-assisted coding faster, more local, and less
token-intensive for everyone.

The public source and official `cdidx` binaries are source-available under the
Functional Source License, Version 1.1, ALv2 Future License
(`FSL-1.1-ALv2`), unless a specific file or directory states another license.

This policy explains the intended boundary in plain language. It does not
replace the license text in `LICENSE` and `LICENSES/FSL-1.1-ALv2.txt`.

## Allowed Without a Separate Agreement

You may use CodeIndex without a separate agreement for ordinary development
and non-competing integration, including commercial use.

Allowed uses include:

- personal software development;
- commercial software development;
- internal company, school, nonprofit, public-sector, or team use;
- searching, indexing, inspecting, and retrieving information from codebases
  you own, control, or are authorized to access;
- using CodeIndex to reduce AI token usage while developing your own products;
- configuring AI coding agents to call a user-installed `cdidx` binary;
- invoking CodeIndex through CLI, JSON output, or MCP for local development;
- using CodeIndex from terminals, scripts, CI workflows, editors, IDEs, and
  developer workstations;
- using CodeIndex in an internal developer platform that is not offered to
  third parties as a code indexing, code search, code retrieval, or MCP code
  context product;
- publishing documentation, examples, wrappers, or client integrations that
  call official CodeIndex releases and do not include a competing implementation;
- redistributing unmodified official CodeIndex source or binaries through
  package managers, mirrors, devcontainers, installation scripts, or internal
  artifact caches, provided required notices are preserved;
- providing professional services to help a licensee install, configure, or
  use CodeIndex in accordance with the license;
- contributing improvements back to this repository.

## Requires a Separate Written Agreement

A separate written agreement with Widthdom is required if you make CodeIndex,
a modified version of CodeIndex, or a derivative work of CodeIndex available
to third parties as a commercial product or service whose primary or substantial
purpose is to provide functionality that substitutes for CodeIndex.

Examples that require a separate agreement include:

- selling a fork or derivative of CodeIndex as a code indexing/search product;
- offering a hosted code search, code indexing, repository understanding, or
  code retrieval service powered by CodeIndex;
- offering a commercial MCP code-indexing server or code-context retrieval
  product based on CodeIndex;
- embedding a modified CodeIndex engine as the core retrieval/indexing feature
  of a commercial AI coding assistant, IDE, developer platform, or SaaS product
  offered to third parties;
- publishing a renamed binary, package, container image, service, extension,
  or plugin that markets itself as a substitute for CodeIndex;
- removing or obscuring license notices required by `LICENSE`.

## Not Restricted

The following are not restricted merely because they are commercial:

- using CodeIndex while building your own commercial application;
- using CodeIndex to analyze your own private or proprietary repositories;
- using CodeIndex in company-internal development workflows;
- using CodeIndex to save AI tokens during product development;
- using an AI agent that calls a locally installed official `cdidx` binary;
- writing an IDE/editor extension that helps users invoke official CodeIndex
  releases, as long as the extension is not itself a competing CodeIndex-derived
  indexing/search/retrieval product;
- publishing client configuration examples for MCP, IDEs, editors, or agents.

previous license remain governed by the terms that applied to those copies.
New releases, source snapshots, binaries, and packages from this repository are
