## English

- **SQLite WAL durability settings are now explicit (#1925)** — writable indexes now use `synchronous=NORMAL`, pin `wal_autocheckpoint=1000`, run passive checkpoints after outer write commits, and expose the resolved values in `status --json` as `db_pragma_settings`.

## 日本語

- **SQLite WAL の durability 設定を明示しました (#1925)** — writable な index は `synchronous=NORMAL` を使い、`wal_autocheckpoint=1000` を固定し、outer write commit 後に passive checkpoint を実行し、解決済みの値を `status --json` の `db_pragma_settings` で公開します。
