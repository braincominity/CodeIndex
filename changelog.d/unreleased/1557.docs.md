## English

- **Documented CLI / MCP-only API surface — no public library API (#1557)** — `INTEGRATION_POLICY.md` gains an `API Surface and Library Use` section that explicitly codifies the stance: the supported, versioned surfaces are the `cdidx` CLI (including its `--json` output) and the `cdidx mcp` JSON-RPC server, the NuGet package is published as a .NET global tool (`PackAsTool=true`), and `public` types on the assembly (for example `CodeIndex.Database.DbReader` and DTOs in `CodeIndex.Models` / `CodeIndex.Database`) are implementation details that may change, move, or become `internal` without a deprecation cycle. `README.md` adds a `Supported Surfaces` pointer in both the English and Japanese sections, `DEVELOPER_GUIDE.md` records the matching design-decision entry, and the `cdidx` `<Description>` in `src/CodeIndex/CodeIndex.csproj` (which becomes the NuGet metadata description) now states explicitly that there is no public library / SDK API. Embedders are directed to depend on the CLI / MCP / JSON surfaces instead of referencing the assembly.
