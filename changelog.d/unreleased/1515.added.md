## English

- **Forward-compatibility readiness audit on `status` (#1515)** — Every full scan, update, and MCP index now stamps the writing CLI's version into `codeindex_meta` (`cdidx_writer_version`), and `status --json` / MCP status payloads expose three new trust fields: `index_writer_version` (the CLI that last wrote the DB), `index_newer_than_reader` (true when any persisted numeric contract stamp — `metadata_target_version_csharp`, `csharp_symbol_name_contract_version`, `sql_graph_contract_version`, `fold_key_version`, `hotspot_family_version_*` — or unknown `PRAGMA user_version` readiness bits exceed the current binary's compiled maximum), and `index_newer_than_reader_reason` (which specific stamps are newer than the reader). `cdidx status` also prints a human-readable `WARN :` line and `IsStatusDegraded` flips to true in this case, so an older `cdidx` re-opening a DB written by a newer `cdidx` degrades loudly with an audit trail instead of silently flipping readiness bits to false against a workspace that other cdidx clients still treat as authoritative.

## 日本語

- **`status` に forward-compatibility 監査を追加 (#1515)** — full scan / update / MCP index の各経路で書き込み時の CLI バージョンを `codeindex_meta` (`cdidx_writer_version`) に stamp し、`status --json` / MCP status payload に 3 つの信頼フィールドを追加しました: `index_writer_version` (最後に DB を書き込んだ CLI のバージョン)、`index_newer_than_reader` (永続化された数値契約 stamp — `metadata_target_version_csharp`、`csharp_symbol_name_contract_version`、`sql_graph_contract_version`、`fold_key_version`、`hotspot_family_version_*` — や `PRAGMA user_version` の未知の readiness ビットが現在のバイナリのコンパイル時 max を超える場合に true)、`index_newer_than_reader_reason` (どの stamp がリーダーより新しいかの内訳)。`cdidx status` は人間向け出力にも `WARN :` 行を表示し、`IsStatusDegraded` も true になります。これにより、新しい `cdidx` で書かれた DB を古い `cdidx` が再オープンしたときに、他の cdidx クライアントから見れば依然 authoritative なワークスペースに対して readiness を黙って false に倒すことなく、監査痕跡付きで明示的に縮退するようになります。
