## English

- **Windows terminal ANSI detection no longer depends on UTF-16 stdout encoding (#1918)** — `--color=auto` now separates redirected/test-capture detection from Windows virtual-terminal capability checks, so modern ConPTY and Windows Terminal sessions are not misclassified by `Console.Out.Encoding`.

## 日本語

- **Windows 端末の ANSI 判定が stdout の UTF-16 encoding に依存しなくなりました (#1918)** — `--color=auto` は redirected / test capture 判定と Windows の virtual-terminal 能力判定を分離し、ConPTY や Windows Terminal が `Console.Out.Encoding` だけで誤分類されないようになりました。
