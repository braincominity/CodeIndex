## English

- **Symbol definition queries now use the file-kind index for language + kind filters (#1933)** — `symbols` queries with both `--lang` and `--kind` now constrain `symbols.file_id` through the `files(lang)` index so SQLite can probe `idx_symbols_file_kind` instead of scanning every symbol of the requested kind.

## 日本語

- **言語 + 種別で絞るシンボル定義検索が file-kind インデックスを使うようになりました (#1933)** — `--lang` と `--kind` を併用する `symbols` クエリは `files(lang)` インデックス経由で `symbols.file_id` を制約し、SQLite が要求された種別の全シンボルを走査せず `idx_symbols_file_kind` を probe できるようになりました。
