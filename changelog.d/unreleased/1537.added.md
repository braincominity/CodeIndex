## English

- **`batch_query` now returns envelope metadata and per-slot elapsed time (#1537)** — The MCP `batch_query` response gains a top-level `metadata` object exposing `total_elapsed_ms`, `success_count`, and `failure_count`, and every slot in `results` now includes `elapsed_ms` plus a compact `args_summary`. The text content also rolls up the success/failure split (e.g. `Executed 3 queries in 12 ms (1 succeeded, 2 failed).`) so agents can detect partial-failure batches and locate slow inner queries (such as a costly `impact_analysis` hidden inside a 10-query batch) without re-issuing each query individually. The existing `count` and `results` fields are unchanged, so previous consumers continue to work.

## 日本語

- **`batch_query` がエンベロープメタデータとスロット毎の所要時間を返すようになりました (#1537)** — MCP `batch_query` のレスポンスに `total_elapsed_ms` / `success_count` / `failure_count` を含む `metadata` オブジェクトをトップレベルで追加し、各スロットには `elapsed_ms` と `args_summary` を含めるようにしました。テキスト本文も成功/失敗件数のサマリ（例: `Executed 3 queries in 12 ms (1 succeeded, 2 failed).`）を含むため、エージェントはバッチ単位の部分失敗の検出や、10 個のバッチに隠れた重い `impact_analysis` のような遅いクエリの特定を、各クエリを個別に再実行することなく行えます。既存の `count` / `results` フィールドは互換のため維持しています。
