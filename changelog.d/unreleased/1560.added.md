## English

- **Opt-in per-(tool, caller) rate limiting for MCP tool calls (#1560)** — `cdidx mcp` now supports a token-bucket throttle keyed by `(tool, caller)` so a misbehaving client cannot exhaust CPU or memory by spamming `batch_query` carrying multiple `search` calls. Disabled by default so single-user stdio sessions are unaffected; enable by setting `CDIDX_MCP_RATE_LIMIT_RPS` (tokens per second) and optionally `CDIDX_MCP_RATE_LIMIT_BURST` (bucket capacity, default = `max(rps, 1)`) on the server process. Caller identity is captured from the `clientInfo.name` (and `version` when present) of the `initialize` request, falling back to `"unknown"`. Over-quota calls return JSON-RPC `-32000` with `data.error_category = "rate_limited"`, `data.tool`, `data.caller`, and `data.retry_after_ms` so clients can back off. Inside `batch_query`, each inner slot is also checked against the inner-tool bucket and over-quota slots surface `error_category` + `retry_after_ms` in the per-slot result.

## 日本語

- **MCP ツール呼び出し向けの (tool, caller) 単位レート制限を opt-in で追加 (#1560)** — `cdidx mcp` に `(tool, caller)` をキーとするトークンバケット型スロットルを追加し、`batch_query` に複数の `search` を詰めて CPU/メモリを枯渇させるような誤動作クライアントを抑止できるようにしました。既定では無効で stdio 単一ユーザーには影響しません。サーバープロセスで `CDIDX_MCP_RATE_LIMIT_RPS`（1 秒あたりのトークン数）と任意で `CDIDX_MCP_RATE_LIMIT_BURST`（バケット容量、既定は `max(rps, 1)`）を設定して有効化します。呼び出し元 ID は `initialize` の `clientInfo.name`（および `version` があれば併記）から取得し、未指定時は `"unknown"` にフォールバックします。超過した呼び出しは JSON-RPC `-32000` を返し、`data.error_category = "rate_limited"`、`data.tool`、`data.caller`、`data.retry_after_ms` を含むためクライアントはバックオフできます。`batch_query` 内部の各スロットも内側ツールのバケットで判定され、超過スロットは `error_category` と `retry_after_ms` をスロット単位で返します。
