## English

- **Incremental indexing now detects branch / HEAD changes and recommends `--rebuild` (#1508)** — `cdidx index <projectPath>` records the current git HEAD at the end of every successful full scan and compares it on subsequent runs. When the recorded HEAD differs from the workspace HEAD, the CLI emits a `head_changed` warning (also exposed as `head_changed`, `prior_indexed_head_commit`, `current_head_commit`, and `head_change_notice` in `--json` output, and through `workspace_check.head_changed` on `cdidx status --check`) so users know that a default incremental scan after `git switch <branch>` may leave stale rows for branch-specific files. Partial update modes (`--commits` / `--files`) deliberately preserve the captured HEAD so the staleness signal survives until a real full scan reindexes the worktree.

## 日本語

- **Incremental indexing がブランチ / HEAD 切り替えを検知して `--rebuild` を提案するようになりました (#1508)** — `cdidx index <projectPath>` は full scan 成功時に現在の git HEAD を記録し、次回以降の実行で比較します。記録された HEAD と worktree の HEAD が異なる場合、CLI は `head_changed` 警告を表示し、`--json` 出力では `head_changed` / `prior_indexed_head_commit` / `current_head_commit` / `head_change_notice` を、`cdidx status --check` では `workspace_check.head_changed` を併せて公開します。これにより `git switch <branch>` 後の既定の incremental scan がブランチ固有ファイルの古い行を残しうることを利用者が把握できます。`--commits` / `--files` の部分更新は記録 HEAD を意図的に維持し、次の full scan が worktree を再インデックスするまで stale 通知が継続するようにしました。
