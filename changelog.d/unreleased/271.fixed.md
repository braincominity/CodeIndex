## English

- **Rust turbofish call sites now keep their method references (#271)** — `ReferenceExtractor` now accepts an optional `::` between a call name and its generic arguments, so Rust forms like `value.parse::<i32>()` continue to emit `call` rows instead of silently dropping the reference. Added a regression test for a turbofish method call alongside a plain non-turbofish baseline, and documented the Rust language row in `DEVELOPER_GUIDE.md`. Fixes #271.

## 日本語

- **Rust の turbofish 呼び出しでも method reference が落ちなくなりました (#271)** — `ReferenceExtractor` が呼び出し名と generic 引数の間にある `::` を許容するようになり、`value.parse::<i32>()` のような Rust 形でも `call` 行を silent drop せず出力します。turbofish の method call と通常の非 turbofish baseline を確認する regression test を追加し、`DEVELOPER_GUIDE.md` の Rust 行も更新しました。Fixes #271.
