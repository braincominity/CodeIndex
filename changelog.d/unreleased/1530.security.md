## English

- **MCP error responses no longer echo raw `Exception.Message`; `CDIDX_DEBUG=unsafe` now requires an explicit per-process opt-in (#1530)** — Two related leak paths are closed. (1) `McpServer` previously surfaced raw `ex.Message` strings in `tools/call` error responses and the JSON-RPC loop catch-all, so a SQLite or formatter exception that included indexed file content (e.g. SQL fragments quoting source bytes) was forwarded verbatim to the AI client. The new `BuildSanitizedToolErrorMessage` / `BuildSanitizedLoopErrorMessage` helpers return only the tool name (when known) plus the exception type so clients can branch on retry strategy without ingesting indexed text. The full message and stack remain in stderr via `BuildToolErrorLog` / `BuildUnhandledLoopErrorLog` for local debugging. (2) `DbDebug` previously activated raw row dumps whenever `CDIDX_DEBUG=unsafe` was set, so a stale shell-profile or CI environment variable could silently expose indexed source bytes on stderr. The unsafe path now also requires the new `--debug-unsafe` CLI flag (consumed by `ProgramRunner.TryConsumeDebugUnsafeFlag`, with `--` query-escape preserved); env-var-only `unsafe` falls back to redacted mode and emits a one-shot stderr warning. `CDIDX_DEBUG=1` redacted mode is unchanged.

## 日本語

- **MCP のエラー応答に生の `Exception.Message` を埋め込まないようにし、`CDIDX_DEBUG=unsafe` をコマンドラインフラグでの明示 opt-in 必須に変更しました (#1530)** — 関連する 2 つの漏洩経路を塞ぎます。(1) `McpServer` は `tools/call` のエラー応答および JSON-RPC ループの catch-all で生の `ex.Message` 文字列を返していたため、ソースバイトを引用する SQLite やフォーマッターの例外メッセージが AI クライアントへそのまま転送されていました。新たに追加した `BuildSanitizedToolErrorMessage` / `BuildSanitizedLoopErrorMessage` は、ツール名（既知の場合）と例外型のみを返し、クライアントはインデックス済みテキストを取り込まずにリトライ戦略を分岐できます。詳細メッセージとスタックは従来どおり `BuildToolErrorLog` / `BuildUnhandledLoopErrorLog` 経由で stderr に残るため、ローカルでのデバッグは可能です。(2) `DbDebug` は `CDIDX_DEBUG=unsafe` だけで生の行ダンプを有効化していたため、シェルの profile や CI に残った古い環境変数だけでサイレントに索引済みソースバイトが stderr に流れてしまう余地がありました。unsafe 経路は新たに `--debug-unsafe` CLI フラグ（`ProgramRunner.TryConsumeDebugUnsafeFlag` が `--` クエリエスケープを尊重しつつ取り除く）も必要となり、環境変数のみの `unsafe` は redacted モードへ降格し、一度だけ stderr に警告を出します。`CDIDX_DEBUG=1` の redacted モードは従来どおりです。
