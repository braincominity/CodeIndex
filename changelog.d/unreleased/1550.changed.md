## English

- **`cdidx --version` now stamps commit SHA, build date, and clean/dirty flag so dev builds are distinguishable from tagged releases (#1550)** — The human output is one line: `cdidx v<ver> (commit <sha>, built <yyyy-mm-dd>, <clean|dirty>)`. The new `cdidx --version --json` form emits `{name, version, commit, build_date, dirty}` for support tooling and bug-report templates. Metadata comes from an MSBuild target (`StampCdidxBuildMetadata`) that runs `git rev-parse --short=7 HEAD` and `git diff-index --quiet HEAD --` at build time and writes the values into `AssemblyMetadataAttribute`s, so no new runtime dependencies are introduced and reflection-trim builds keep working through the existing `CliJsonSerializerContext`. When the stamp is absent (legacy assemblies, mocked test binaries) `FormatVersionLine` falls back to bare `cdidx v<ver>`, preserving the previous shape. The `install.sh` reinstall validator was tightened to accept either the bare form or the parenthesised metadata suffix only — trailing diagnostic text without parens is still rejected so token-enumeration drift in `--version` cannot hide install regressions.

## 日本語

- **`cdidx --version` に commit SHA / ビルド日 / clean|dirty フラグを刻印し、開発ビルドとリリースタグを判別可能にしました (#1550)** — 人間向け出力は 1 行で `cdidx v<ver> (commit <sha>, built <yyyy-mm-dd>, <clean|dirty>)` の形に統一しました。新設の `cdidx --version --json` は `{name, version, commit, build_date, dirty}` を返し、サポート対応やバグ報告テンプレートでの機械処理に利用できます。値は MSBuild ターゲット `StampCdidxBuildMetadata` がビルド時に `git rev-parse --short=7 HEAD` と `git diff-index --quiet HEAD --` を実行し、`AssemblyMetadataAttribute` に刻むため、新しい実行時依存は追加されず、reflection-trim ビルドも既存の `CliJsonSerializerContext` 経由で動作します。刻印が無いアセンブリ（旧バイナリやテスト用モック）では `FormatVersionLine` が従来通り `cdidx v<ver>` を返すフォールバックを残します。`install.sh` の reinstall validator は「bare 形」と「括弧で囲ったメタデータ末尾」のみを許容するよう締め直し、括弧無しの末尾診断文は引き続き弾くことで、`--version` 出力のトークン列ドリフトが install 退行を隠さないようにしました。
