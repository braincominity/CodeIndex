## English

- **Per-deployment MCP tool enablement gate (#1561)** — operators can now restrict which MCP tools `cdidx mcp` advertises and dispatches. Set `CDIDX_MCP_TOOLS_ALLOW` to a comma-separated allowlist (e.g. `CDIDX_MCP_TOOLS_ALLOW=search,references,callers`) to pin the visible/callable surface for a read-only or scoped session, or set `CDIDX_MCP_TOOLS_DENY` (e.g. `CDIDX_MCP_TOOLS_DENY=index,backfill_fold`) to remove individual tools from the default-all-enabled set. Allow wins over deny when both are set. `tools/list` only advertises enabled tools, the `initialize` instructions string no longer recommends disabled tools, and a top-level `tools/call` on a disabled known tool returns the structured `-32601 Tool not enabled: <name>` JSON-RPC error so AI clients can branch on a code instead of parsing prose. The `batch_query` envelope still succeeds, but each disabled-tool slot carries a `code: -32601` field alongside the `error` string for the same branching contract. Truly unknown tool names still surface as the existing `-32602 Unknown tool`, keeping operator-disabled tools distinguishable from typos. Unknown names in either env var are filtered against the known tool set; a typo'd allowlist that names only unknown tools intentionally exposes nothing rather than silently disabling the gate.

## 日本語

- **デプロイ単位での MCP ツール有効化ゲートを追加 (#1561)** — `cdidx mcp` が広告／dispatch する MCP ツールをオペレータが制限できるようになりました。`CDIDX_MCP_TOOLS_ALLOW` にカンマ区切りの allowlist を設定すると（例: `CDIDX_MCP_TOOLS_ALLOW=search,references,callers`）、read-only セッションや scoped セッションの表示・呼び出し集合を固定できます。`CDIDX_MCP_TOOLS_DENY` を設定すると（例: `CDIDX_MCP_TOOLS_DENY=index,backfill_fold`）既定の全有効集合から個別に除外できます。両方指定された場合は allow を優先します。`tools/list` は有効ツールのみ広告し、`initialize` の instructions 文字列も無効化されたツールを推奨しなくなります。トップレベル `tools/call` で無効化された既知ツールを呼び出した場合は構造化された JSON-RPC エラー `-32601 Tool not enabled: <name>` を返すため、AI クライアントは文言の部分一致に頼らず code で分岐できます。`batch_query` 自体は引き続き成功しますが、無効化ツールの各 slot に `code: -32601` フィールドが `error` 文字列と並んで載るので同じ分岐契約を保てます。サーバーに元から無い名前は既存の `-32602 Unknown tool` を返し、オペレータによる無効化と typo を区別可能にしています。いずれの env var の値も既知ツール集合で filter され、typo で未知名のみになった allowlist は意図的に何も公開しないため、ゲートが silent に外れることはありません。
