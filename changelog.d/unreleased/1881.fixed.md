## English

- **`impact` now anchors file-level hints on same-file `call` / `instantiate` evidence (#1881)** — the file-dependency heuristic used by `impact` when no transitive callers exist now recognizes an exact-name `call` or `instantiate` reference to the resolved target as a strictly stronger anchor than the looser metadata-attribute bypass. A file that genuinely instantiates a target type therefore surfaces as a dependent even when the call site's enclosing container is missing and no signature / return-type token mentions the target. The new anchor matches the resolved name exactly with no `Attribute` suffix-strip alias, so unrelated `Foo()` method calls cannot widen `impact FooAttribute` blast radius.

## 日本語

- **`impact` のファイル単位ヒントが同一ファイル内の `call` / `instantiate` 証拠を anchor として認識するようになりました (#1881)** — transitive caller が見つからないときに使われるファイル依存 heuristic は、解決済み target への厳密一致の `call` / `instantiate` 参照を、より緩い metadata-attribute bypass より強い anchor として扱います。これにより、enclosing container が欠落していて signature / return 型に target 名が出現しないケースでも、その型を実際に instantiate しているファイルが依存元として現れます。新しい anchor は `Attribute` サフィックスの strip 別名を使わない厳密一致のため、無関係な `Foo()` 呼び出しが `impact FooAttribute` の影響範囲を不当に広げることはありません。
