## English

- **`hotspots` now ranks lower-signal call-graph edges below real invocations (#1875)** — hotspot ordering uses a weighted invocation score so `subscribe` edges no longer outrank fewer direct `call` / `instantiate` sites solely by raw count, while human and JSON output still expose the raw reference count.

## 日本語

- **`hotspots` は低シグナルな call-graph edge を実呼び出しより下に順位付けするようになりました (#1875)** — hotspot の順位付けに重み付き invocation score を使うため、`subscribe` edge が raw count だけで少数の直接 `call` / `instantiate` site を上回らなくなりました。human / JSON 出力では引き続き raw reference count も確認できます。
