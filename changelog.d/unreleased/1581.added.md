## English

- **Structured error envelope and server-defined codes for the MCP server (#1581)** — `cdidx mcp` now returns a canonical `data` envelope (`category`, `suggestion`, `retry_safe`) on every JSON-RPC `error` object and every MCP tool-result error (`isError: true`), so AI clients can branch on a stable machine-readable category instead of parsing the human `message` string. Server-defined JSON-RPC codes are allocated within the reserved `-32000..-32099` range: `-32000 rate_limited`, `-32001 permission_denied`, `-32010 index_missing`, `-32011 index_stale`, `-32012 index_corrupted`, `-32015 request_cancelled`. Standard JSON-RPC codes (`-32700` / `-32600..-32603`) keep their semantics with new categories layered on top (`parse_error`, `invalid_request`, `message_too_large`, `method_not_found`, `tool_unknown`, `tool_disabled`, `missing_parameter`, `invalid_argument`, `internal_error`). Unhandled exceptions inside `tools/call` go through `McpErrorEnvelope.ClassifyException`, which maps `SqliteException` to `index_stale` (missing table/column) or `index_corrupted` (malformed / not-a-database / encrypted), `OperationCanceledException` to `request_cancelled`, and everything else to `internal_error`; the raw exception message is never leaked on the wire (per #1530 sanitization). Legacy fields on rate-limited responses (`error_category`, `tool`, `caller`, `retry_after_ms` from #1560) are preserved alongside the new canonical envelope so existing clients keep working. The full code/category map is documented in `DEVELOPER_GUIDE.md`.

## 日本語

- **MCP サーバーに構造化エラーエンベロープとサーバー定義コードを追加 (#1581)** — `cdidx mcp` がすべての JSON-RPC `error` オブジェクトと MCP ツール結果エラー（`isError: true`）に対し、canonical な `data` エンベロープ（`category` / `suggestion` / `retry_safe`）を必ず返すようになりました。これにより AI クライアントは `message` 文字列を解析せず、安定した機械可読カテゴリで分岐できます。サーバー定義 JSON-RPC コードは予約レンジ `-32000..-32099` 内で割り当て: `-32000 rate_limited` / `-32001 permission_denied` / `-32010 index_missing` / `-32011 index_stale` / `-32012 index_corrupted` / `-32015 request_cancelled`。標準コード（`-32700` / `-32600..-32603`）は意味を保ったまま新カテゴリ（`parse_error` / `invalid_request` / `message_too_large` / `method_not_found` / `tool_unknown` / `tool_disabled` / `missing_parameter` / `invalid_argument` / `internal_error`）を載せます。`tools/call` 内の未処理例外は `McpErrorEnvelope.ClassifyException` を通り、`SqliteException` を `index_stale`（テーブル/カラム欠落）または `index_corrupted`（malformed / not-a-database / encrypted）に、`OperationCanceledException` を `request_cancelled` に、その他を `internal_error` にマッピングします。生の例外メッセージは引き続きワイヤに乗りません（#1530 のサニタイズ方針に従う）。rate-limited 応答の legacy フィールド（#1560 の `error_category` / `tool` / `caller` / `retry_after_ms`）は canonical エンベロープと併載して既存クライアントを壊しません。コード・カテゴリ表の全体は `DEVELOPER_GUIDE.md` に記載しています。
