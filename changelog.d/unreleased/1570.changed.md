## English

- **CLI flags now live in a single schema instead of being duplicated across help text, parser allowlists, and shell completion templates (#1570)** — `CliFlagSchema.cs` is the new source of truth: each flag declares its `Commands` (parser + completion) and `AlsoAcceptedBy` (parser-only, hidden from completion, e.g. `--exact-name` on `search`). `QueryCommandRunner` now drives every `TryWriteUnsupportedOptionError` allowlist and `ValidateFindArgs` from this schema, and `ConsoleUi` generates the bash / zsh / fish completion scripts from it too. Adding a new flag is now a one-row change instead of four hand-synchronized edits. `CliFlagSchemaTests` guards the SSoT contract (parity between schema and `ConsoleUi.Commands`, no flag references an unknown subcommand, completion scripts contain every schema flag and nothing extra, parser-vs-completion drift is intentional). Side effects: per-branch `--max-line-width` tooltips (previously varied between `snippets` / `contexts` / `excerpts`) now use a single canonical "payloads" wording.

## 日本語

- **CLI フラグを単一スキーマに統合し、ヘルプテキスト・パーサ allowlist・シェル補完テンプレートの三重管理を解消しました (#1570)** — `CliFlagSchema.cs` が新しい単一情報源となり、各フラグは `Commands`（パーサと補完の両方で受理）と `AlsoAcceptedBy`（パーサは受理するが補完では出さない／`search` 上の `--exact-name` のようなケース）を宣言します。`QueryCommandRunner` 内のすべての `TryWriteUnsupportedOptionError` 許可リストと `ValidateFindArgs` はこのスキーマから生成され、`ConsoleUi` の bash / zsh / fish 補完スクリプトも同じくスキーマ駆動になりました。新規フラグの追加が 4 箇所同期から 1 行追加に短縮されます。`CliFlagSchemaTests` がスキーマと `ConsoleUi.Commands` の整合、未知サブコマンド参照の検出、補完に schema 外フラグが混入していないこと、parser-vs-completion の意図的な乖離 を CI で検査します。副次的な変更として、`--max-line-width` のブランチ別ツールチップ（旧 `snippets` / `contexts` / `excerpts`）は統一表記の "payloads" になります。
