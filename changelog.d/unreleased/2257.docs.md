## English

- **Branch-switch indexing guidance now prefers `--changed-between` (#2257)** — agent and user documentation now tell branch-switch workflows to refresh with `cdidx index <projectPath> --changed-between <old-ref> <new-ref>` when both refs are known, reserving full scans for history-moving operations, missing refs, or repo-wide stale-path purges.

## 日本語

- **ブランチ切り替え後の index 手順で `--changed-between` を優先するよう明確化しました (#2257)** — agent 向けおよびユーザー向けドキュメントは、切り替え前後の ref が分かる場合に `cdidx index <projectPath> --changed-between <old-ref> <new-ref>` で更新し、履歴を動かす操作、ref 不明時、または repo 全体の stale path purge が必要な場合だけ full scan を使うよう案内します。
