## English

- **`--json-envelope` wraps `--json` output in a `{ "metadata": {...}, "results": [...] }` object (#1527)** — query commands (`search`, `definition`, `callers`, `callees`, `impact`, `symbols`, `find`, `status`, etc.) accept the new `--json-envelope` flag. When set, the wrapper captures the per-result NDJSON stream into a single JSON document whose `metadata` carries `command`, `cdidx_version`, `elapsed_ms`, `db_path`, `result_count`, `exit_code`, and (when applicable) `query_normalized` and `indexed_at_head_sha`. The legacy flat NDJSON / array output remains the default for one release so existing scripts keep working; the envelope will become the default in the next major version, and the flat form will then be opt-in via `--json-flat`.

## 日本語

- **`--json-envelope` で `--json` 出力を `{ "metadata": {...}, "results": [...] }` 形式にラップ (#1527)** — `search` / `definition` / `callers` / `callees` / `impact` / `symbols` / `find` / `status` など各クエリコマンドに `--json-envelope` フラグを追加しました。指定すると per-result NDJSON ストリームを 1 つの JSON ドキュメントにまとめ、`metadata` に `command`、`cdidx_version`、`elapsed_ms`、`db_path`、`result_count`、`exit_code`、および該当する場合は `query_normalized`、`indexed_at_head_sha` を載せます。後方互換のため従来の flat NDJSON / 配列出力は今回のリリースまで既定のままとし、次のメジャーバージョンで envelope を既定に切り替え、flat 形式は `--json-flat` でオプトインに移行する予定です。
