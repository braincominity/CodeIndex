## English

- **Warn when the worktree HEAD has switched since the index was built (#1512)** — `status` (CLI text + JSON) and MCP `repo/status` / symbol-analysis responses now compare the persisted `indexed_head_commit` (recorded at the end of a successful full-scan index — see #1508) against the runtime worktree HEAD without needing a `--check` workspace scan. When the two diverge — for example after `git worktree add` or `git switch` inside an existing worktree — the new top-level `indexed_head_commit` / `worktree_head_changed` JSON fields surface the drift, and the CLI text mode prints a `WARN` line with a ready-to-run `cdidx index` command so the index can be refreshed for the current branch. MCP-driven re-indexes mirror the CLI contract and stamp the HEAD only when the run is fully successful, so partial failures keep the prior HEAD and continue to surface staleness until the next clean refresh. Legacy DBs without the persisted HEAD return null fields and trigger no warning, preserving back-compat.

## 日本語

- **worktree の HEAD が index 構築後に切り替わった場合に警告 (#1512)** — `status` (CLI テキストおよび JSON) と MCP の `repo/status` / symbol-analysis 応答が、`--check` を介さずに永続化された `indexed_head_commit` (full-scan 成功時に #1508 で記録) と現在の worktree の runtime HEAD を突き合わせます。`git worktree add` や worktree 内での `git switch` 直後など差分があるときは、トップレベルの `indexed_head_commit` / `worktree_head_changed` フィールドが状態を返し、CLI テキストモードでは現在のブランチで index を更新するための `cdidx index` コマンド付き `WARN` 行を出力します。MCP 経由の再 index も CLI と同じ規約に揃え、成功時のみ HEAD を保存することで、partial 失敗時は旧 HEAD を残し次回 full scan まで stale を表面化させ続けます。persisted HEAD を持たない旧 DB ではフィールドは null となり警告も出ないので、後方互換を維持します。
