## English

- **Numeric CLI flags now enforce per-flag upper bounds (#1503)** — `--limit` / `--top` (≤ 10000), `--snippet-lines` (≤ 20, matching the MCP `search` tool schema), `--depth` (≤ 64), `--before` / `--after` (≤ 1000), `--start` / `--end` / `--focus-line` (≤ 10000000), `--focus-column` / `--focus-length` (≤ 100000) now reject values above the documented maximum with `Error: <flag> must be less than or equal to <max>, got '<value>'.` instead of silently parsing `int.MaxValue` (which previously produced surprisingly large allocations / output) or silently clamping (`--snippet-lines 999999` previously folded down to 20 with no signal). The `--max-line-width` ceiling check is now handled by the same shared upper-bound table. `find` validation rejects out-of-range `--limit` / `--before` / `--after` before reaching the main parser.
