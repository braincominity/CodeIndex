## English

- **`InsertReferences` no longer leaks orphan `reference_lines` rows when a chunk fails under an outer transaction (#1518)** — the chunked batch writer used to skip its inner transaction when nested under a caller-owned transaction, so a mid-chunk `symbol_references` INSERT failure (e.g. trigger ABORT or FK violation) left the matching `reference_lines` UPSERTs visible to the outer transaction. They were then committed once the outer transaction committed, accumulating as orphan rows that only `--rebuild` could clean. Each batch now opens a chunk-scoped transaction or `SAVEPOINT` so both inserts share a single rollback boundary regardless of outer transaction state.

## 日本語

- **`InsertReferences` で外側トランザクション配下のチャンク失敗時に `reference_lines` の孤児行が発生する問題を修正しました (#1518)** — バッチ書き込みは呼び出し側のトランザクションがあると内側のトランザクションを張らない実装だったため、チャンク途中で `symbol_references` の INSERT が失敗（トリガー ABORT や外部キー違反など）すると、同じチャンクで UPSERT 済みの `reference_lines` が外側トランザクションに残り、外側コミット時に孤児行として残留していました。これは `--rebuild` でしか掃除されません。各チャンクで常にトランザクションまたは `SAVEPOINT` を張り、両 INSERT が同じロールバック境界を共有するようにしました。
