## English

- **`status` reports indexed HEAD metadata and cross-session drift (#1509)** — `cdidx index` now stamps `indexed_head_sha`, `indexed_head_branch`, and `indexed_head_timestamp` into `codeindex_meta` on a successful run, and `status` (human + `--json` + MCP) surfaces them alongside a new `commits_ahead_of_indexed_head` count, so a stale index ("indexed at commit X, you are now N commits ahead") is detectable across sessions. Non-git workspaces and detached HEAD are handled gracefully (branch omitted, SHA still stamped when available), and stamping is best-effort so a transient `git` failure never fails an otherwise-successful index.

## 日本語

- **`status` がインデックス時 HEAD メタデータと cross-session ドリフトを報告するようになりました (#1509)** — `cdidx index` 成功時に `indexed_head_sha` / `indexed_head_branch` / `indexed_head_timestamp` を `codeindex_meta` に記録し、`status`（human / `--json` / MCP）が新規 `commits_ahead_of_indexed_head` カウントと併せて表示するため、「commit X 時点のインデックスで現在 N コミット先行」というセッション跨ぎの古さを検出できます。非 git ワークスペースや detached HEAD でも壊れず（ブランチは省略し、可能なら SHA だけ記録）、stamping は best-effort のため一過性の `git` 失敗がインデックス成功を阻害することはありません。
