## English

- **Stable machine-readable error-code taxonomy on CLI / MCP failures (#1526)** — `cdidx` errors now carry a stable `Exxx_NAME` code so scripts, oncall runbooks, and AI agents can classify failures without substring-matching the human-readable prose. Human stderr output uses the bracketed prefix `Error [E001_DB_NOT_FOUND]: database not found at …` and `--json` envelopes gain an additive optional `error_code` field (omitted when null, so existing consumers see no schema break). Codes wired in this release: `E001_DB_NOT_FOUND` (db file missing), `E002_DB_LOCKED` (SQLite BUSY/LOCKED or per-database file-lock contention), `E003_SCHEMA_TOO_NEW` (reserved — currently surfaced softly via `status --json index_newer_than_reader`), `E004_DB_NOT_WRITABLE`, `E005_DB_INTEGRITY_FAILED` (`PRAGMA integrity_check` rows), `E006_FTS_QUERY_SYNTAX` (raw `--fts` parse error), `E007_TEMP_STORE_EXHAUSTED` (SQLite `SQLITE_FULL`), `E008_DB_ERROR` (generic SQLite fallback), `E009_FEATURE_UNAVAILABLE` (e.g. `--json` on a trimmed/AOT build), `E010_USAGE_ERROR` (argument-parse / conflicting flags), and `E011_DIRECTORY_NOT_FOUND` (project / target directory missing). Once published, codes will not be renamed or reused — retired codes stop emitting but the constants stay in `CommandErrorCodes.cs`. See `DEVELOPER_GUIDE.md` § *Error code taxonomy* for the full table and stability contract.

## 日本語

- **CLI / MCP のエラー出力に安定した機械可読エラーコードを付与 (#1526)** — `cdidx` のエラーに安定した `Exxx_NAME` コードが付くようになり、スクリプト・オンコール runbook・AI エージェントが人間向け文言の部分一致なしで失敗を分類できるようになりました。人間向けの stderr では `Error [E001_DB_NOT_FOUND]: database not found at …` のように角括弧で前置し、`--json` エンベロープには追加の任意フィールド `error_code` が付きます（null の場合は省略されるので、既存の JSON 利用者にはスキーマ破壊なし）。本リリースで割り当てたコード: `E001_DB_NOT_FOUND`（DB ファイル不在）、`E002_DB_LOCKED`（SQLite BUSY/LOCKED または DB 単位の file-lock 競合）、`E003_SCHEMA_TOO_NEW`（予約 — 現状は `status --json index_newer_than_reader` でソフトに表示）、`E004_DB_NOT_WRITABLE`、`E005_DB_INTEGRITY_FAILED`（`PRAGMA integrity_check` の検出行）、`E006_FTS_QUERY_SYNTAX`（生 `--fts` のパース失敗）、`E007_TEMP_STORE_EXHAUSTED`（SQLite `SQLITE_FULL`）、`E008_DB_ERROR`（汎用 SQLite フォールバック）、`E009_FEATURE_UNAVAILABLE`（trimmed/AOT ビルドでの `--json` など）、`E010_USAGE_ERROR`（引数パース / フラグ競合）、`E011_DIRECTORY_NOT_FOUND`（プロジェクト / 対象ディレクトリ不在）。一度公開したコードは renaming / 使い回しをせず、廃止する場合も新規 emission を止めるだけで定数は `CommandErrorCodes.cs` に残します。完全な一覧と安定性契約は `DEVELOPER_GUIDE.md` の *Error code taxonomy* 節を参照してください。
