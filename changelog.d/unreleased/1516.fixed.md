## English

- **`TryMigrateForRead` now surfaces partial-DDL failures instead of swallowing them (#1516)** — when a schema migration step fails on a restricted mount (read-only filesystem, sandbox, WORM / network share), `DbContext.TryMigrateForRead` now records the failing step, the SQLite error code, and an actionable suggestion on a new `LastMigrationFailure` property, and emits a single `Warning: cdidx schema migration step "<step>" failed (SQLite error <code>: ...)` line to `stderr` pointing the user at writable-storage recovery (e.g. `chmod +w` on the `.cdidx` directory). This replaces the previous broad swallow that left users with cryptic later `no such column` errors and no way to tell whether the cause was permissions, disk space, or corruption.

## 日本語

- **`TryMigrateForRead` が部分 DDL 失敗を握り潰さず通知するようになりました (#1516)** — 読み取り専用 FS / sandbox / WORM・ネットワーク共有といった制限付きマウント上でスキーマ移行ステップが失敗した場合、`DbContext.TryMigrateForRead` は失敗したステップ名・SQLite エラーコード・推奨アクションを新しい `LastMigrationFailure` プロパティに記録し、`Warning: cdidx schema migration step "<step>" failed (SQLite error <code>: ...)` を 1 行で `stderr` に出力して書き込み可能ストレージへの誘導（`.cdidx` ディレクトリへの `chmod +w` 等）を提示するようになりました。これまで広範な握り潰しのために後続クエリが解読困難な `no such column` で失敗し、原因がパーミッションかディスク容量かデータ破損か切り分けできなかった問題が解消します。
