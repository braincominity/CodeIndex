## English

- **Scanner now skips macOS AppleDouble (`._*`) resource forks by default (#1583)** — both the full-scan walker and the update-mode path filter (`--files` / `--commits`) now treat any filename starting with `._` as a default-excluded file, the same way `.DS_Store` / `Thumbs.db` are already handled, so AppleDouble metadata files no longer pass through the indexability probe pretending to be the real file's language and no longer feed binary blobs to the symbol extractor. Recognized dotfiles (`.gitignore`, `.editorconfig`, `.cdidxrc.json`, …) are not affected because they do not start with the `._` AppleDouble prefix. Users who need to suppress additional files can still layer rules in `.gitignore` or `.cdidxignore`.

## 日本語

- **scanner が macOS AppleDouble (`._*`) を既定で除外 (#1583)** — full-scan walker と `--files` / `--commits` の update-mode path filter のいずれも、ファイル名が `._` で始まるエントリを `.DS_Store` / `Thumbs.db` と同様の既定除外対象として扱うようになりました。これにより AppleDouble メタデータがインデックス可能性プローブを通過して原ファイルの言語に偽装したり、シンボル抽出器がバイナリ blob を解析対象にしたりすることがなくなります。`.gitignore` / `.editorconfig` / `.cdidxrc.json` のような既知 dotfile は `._` 接頭辞を持たないため影響を受けません。追加の除外が必要な場合は引き続き `.gitignore` や `.cdidxignore` でルールを重ねられます。
