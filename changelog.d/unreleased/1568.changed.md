## English

- **CLI easter-egg messages now respect a user-facing language setting (#1568)** — bilingual user-facing strings in `ConsoleUi.PrintEasterEggMessage` are no longer inlined as raw English+Japanese pairs. The new `UiMessages` catalog (in `src/CodeIndex/Cli/UiI18n.cs`) co-locates each translation, and `UiLanguageResolver` picks the active language with precedence `CDIDX_LANG` env (`en` / `ja` / `both` and common aliases) > `CultureInfo.CurrentUICulture` (`ja-*` → Japanese) > English fallback. **Default-output change:** the easter-egg banners (`cdidx --sushi`, `--coffee`, `--ramen`, `--wine`, `--beer`, `--matcha`, `--whisky`) previously always printed both an English line and a Japanese line; they now print only the resolver-selected language. Set `CDIDX_LANG=both` (or `bilingual`) to restore the previous always-bilingual output. This is the first step in extracting all user-facing strings from `ConsoleUi` (and, later, `ProgramRunner`) into a single i18n catalog so new translations no longer require editing every call site.

## 日本語

- **CLI のイースターエッグ表示がユーザー言語設定を尊重するようになりました (#1568)** — `ConsoleUi.PrintEasterEggMessage` の二言語ユーザー向け文字列を、英語と日本語のペアでベタ書きするのをやめました。新しい `UiMessages` カタログ (`src/CodeIndex/Cli/UiI18n.cs`) が各翻訳を 1 か所にまとめ、`UiLanguageResolver` が `CDIDX_LANG` 環境変数 (`en` / `ja` / `both` および主要なエイリアス) > `CultureInfo.CurrentUICulture` (`ja-*` → 日本語) > 英語フォールバック の優先順で表示言語を決定します。**既定出力の変更:** `cdidx --sushi` / `--coffee` / `--ramen` / `--wine` / `--beer` / `--matcha` / `--whisky` のイースターエッグは、これまで常に英語行と日本語行の 2 行を出力していましたが、本変更以降は解決された言語のみを出力します。従来の二言語出力に戻したい場合は `CDIDX_LANG=both`（または `bilingual`）を設定してください。`ConsoleUi`（および将来的に `ProgramRunner`）のユーザー向け文字列を共通 i18n カタログへ移行する第一歩で、翻訳追加時に各呼び出し元を書き換えずに済むようにします。
