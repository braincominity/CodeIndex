## English

- **`--color {auto,always,never}` flag and `NO_COLOR` / `CLICOLOR_FORCE` / `CLICOLOR` env support (#1524)** — `cdidx` previously suppressed ANSI color whenever stdout was redirected, so common pager workflows such as `cdidx symbols Foo | less -R` lost color even though the pager would render it. The CLI now accepts a global `--color` flag and a documented env precedence: the flag wins over env vars, `NO_COLOR` (any non-empty value) disables color, `CLICOLOR_FORCE=1` forces it on even when stdout is redirected, `CLICOLOR=0` disables it, and otherwise the existing TTY auto-detect path is preserved. Invalid `--color` values are rejected with a usage error and a hint listing the supported values.

## 日本語

- **`--color {auto,always,never}` フラグと `NO_COLOR` / `CLICOLOR_FORCE` / `CLICOLOR` 環境変数のサポートを追加 (#1524)** — `cdidx` は stdout がリダイレクトされると常に ANSI カラーを抑止していたため、`cdidx symbols Foo | less -R` のような pager パイプでは pager 側が色を解釈できるにもかかわらず色が失われていました。CLI でグローバルな `--color` フラグを受け付け、優先順位を明文化しました: フラグが環境変数より優先、`NO_COLOR`（空でない任意の値）で色を無効化、`CLICOLOR_FORCE=1` でリダイレクト時でも色を強制、`CLICOLOR=0` で無効化、いずれも未設定なら既存の TTY 自動判定にフォールバック。`--color` に未知の値が渡された場合は使用法エラーと許容値のヒントを表示します。
