## English

- **Ruby block-call forms now appear in `references`, `callers`, `callees`, and `impact` (#278)** — `ReferenceExtractor` now recognizes idiomatic Ruby block-call syntax such as `items.each { ... }`, `items.each_with_index do ... end`, and `with_transaction do ... end`, while keeping hash literals from being mistaken for calls.

## 日本語

- **Ruby の block-call 形が `references` / `callers` / `callees` / `impact` に出るようになりました (#278)** — `ReferenceExtractor` が `items.each { ... }`、`items.each_with_index do ... end`、`with_transaction do ... end` のような慣用的な Ruby block-call 構文を認識するようになり、hash literal を呼び出しとして誤認しないようになりました。
