## English

- **`api_version` field on top-level CLI/MCP JSON DTOs (#1555)** — every top-level CLI/MCP JSON output DTO (`StatusResult`, `RepoMapResult`, `SymbolAnalysisResult`, `ImpactAnalysisResult`, `OutlineResult`, `FileExcerptResult`, `CompactSearchResult`, `SymbolResult`, `DefinitionResult`, `UnusedSymbolResult`, `ReferenceResult`, `CallerResult`, `CalleeResult`, `FileResult`, `FileFindResult`) now carries an `api_version` string field stamped from the new public `JsonOutputContract.ApiVersion` constant, and `--json-envelope` mirrors the same value on its `metadata` block. This describes the JSON output contract, not the cdidx binary version, so strict downstream consumers can pin against a known shape and detect breaking renames/removals/type-changes (the version is bumped only on those, not on additive fields).

## 日本語

- **CLI/MCP のトップレベル JSON DTO に `api_version` フィールドを追加 (#1555)** — CLI/MCP のトップレベル JSON 出力 DTO（`StatusResult`、`RepoMapResult`、`SymbolAnalysisResult`、`ImpactAnalysisResult`、`OutlineResult`、`FileExcerptResult`、`CompactSearchResult`、`SymbolResult`、`DefinitionResult`、`UnusedSymbolResult`、`ReferenceResult`、`CallerResult`、`CalleeResult`、`FileResult`、`FileFindResult`）はすべて、新規追加した public 定数 `JsonOutputContract.ApiVersion` から stamp された `api_version` 文字列フィールドを返すようになりました。`--json-envelope` の `metadata` ブロックも同じ値をミラーします。これは JSON 出力契約のバージョンであり、cdidx バイナリのバージョンとは別物です。strict な downstream 利用者は既知のシェイプに pin でき、フィールド名の rename / 削除 / 型変更のような破壊的変更を検出できます（追加のみの変更ではバージョンは据え置かれます）。
