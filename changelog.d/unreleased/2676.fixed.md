## English

- **Ready-bit stamping no longer starts nested SQLite transactions on shared writers (#2676)** — `SetReadyBit` and fold-ready stamping now share the writer transaction gate, so concurrent calls wait for an active scope before issuing `BEGIN IMMEDIATE`.

## 日本語

- **ready-bit stamp が共有 writer 上で nested SQLite transaction を開始しないようにしました (#2676)** — `SetReadyBit` と fold-ready stamp は writer transaction gate を共有し、active scope が終わるまで待ってから `BEGIN IMMEDIATE` を発行します。
