## English

- **Optional HTTP transport for the MCP server (#1558)** — `cdidx mcp` now accepts `--transport stdio|http` (default `stdio`) plus `--http-listen <host:port>` (default `127.0.0.1:38080`). The HTTP transport accepts one JSON-RPC frame per POST and returns the matching response in the same HTTP body (notifications return `204 No Content`), keeping the request/response ordering invariant the existing stdio loop relies on. The listener refuses to bind to non-loopback hosts unless a shared secret is supplied via the `CDIDX_MCP_HTTP_TOKEN` environment variable, in which case every request must carry a matching `Authorization: Bearer ...` header. Stdio behavior is byte-for-byte unchanged, so existing clients continue to work without modification.

## 日本語

- **MCP サーバー向けのオプション HTTP トランスポート (#1558)** — `cdidx mcp` に `--transport stdio|http`（既定 `stdio`）と `--http-listen <host:port>`（既定 `127.0.0.1:38080`）を追加しました。HTTP トランスポートは POST 1 件あたり JSON-RPC フレーム 1 件を受け取り、同じ HTTP レスポンスのボディに応答を返します（通知は `204 No Content`）。これにより既存の stdio ループが依存する「リクエスト 1 件 → レスポンス 1 件」の順序不変条件が維持されます。listener は loopback 以外の host にバインドする際、`CDIDX_MCP_HTTP_TOKEN` 環境変数で共有秘密が指定されていなければ起動を拒否し、指定されている場合はすべてのリクエストに `Authorization: Bearer ...` ヘッダーが必要となります。stdio の挙動はバイト単位で変更ないため、既存クライアントはそのまま動作します。
