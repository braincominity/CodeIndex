## English

- **Documented the reference_kind filtering matrix for graph entry points (#1882)** — `DEVELOPER_GUIDE.md` now includes a bilingual matrix that lists every graph entry point (`references`, `callers` / `callees`, `impact` callers mode, `impact` file-hint fallback, `deps` forward, `deps --reverse`) alongside the `reference_kind` subset it walks and the backing `DbReader` function. The matrix makes the intentional call-graph vs. dependency-graph split explicit — call-graph commands strip `attribute` / `annotation` via `CallGraphReferenceKindsSql` while `deps` keeps them as compile-time edges, and `deps` forward / reverse share a single SQL function so the two directions cannot drift apart. The reconciliation path (`references --kind attribute`) is also documented, and a new regression test `DbReaderTests.ReferenceKindMatrix_DepsKeepsMetadataInBothDirections_CallersExcludesMetadata` pins the contract end-to-end.
