## English

- **`--exact`, `--exact-substring`, and `--exact-name` are now mutually exclusive (#1504)** — `QueryCommandRunner` rejects any combination of two or more of the three exactness flags with `Error: pass only one of --exact, --exact-substring, --exact-name.` instead of silently OR-merging them. `search` / `definition` / `references` / `callers` / `callees` / `symbols` / `inspect` all share the same parse-time check, and the CLI `--help` line plus `USER_GUIDE.md` document the constraint so the precedence is no longer undefined.

## 日本語

- **`--exact` / `--exact-substring` / `--exact-name` を相互排他に変更 (#1504)** — `QueryCommandRunner` は 3 つの厳密一致フラグのうち 2 つ以上を同時に渡された場合、暗黙の OR 合成をやめて `Error: pass only one of --exact, --exact-substring, --exact-name.` で usage error として拒否するようになりました。`search` / `definition` / `references` / `callers` / `callees` / `symbols` / `inspect` のいずれでも同じ parse 時点チェックが適用され、CLI `--help` と `USER_GUIDE.md` にも制約を明記したので、優先順位が未定義のまま振る舞いがずれることがなくなります。
