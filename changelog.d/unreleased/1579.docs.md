## English

- **USER_GUIDE.md adds a Troubleshooting section cross-referenced to the error-code taxonomy (#1579)** — `USER_GUIDE.md` now has a `## Troubleshooting` section (English + 日本語) cataloging 15 common failure modes — missing/stale/locked/non-writable/malformed databases, disk-full / temp-store exhaustion, schema-too-new, stale fold metadata, FTS5 syntax errors, non-UTF-8 / permission-denied / oversize skips, trimmed-build feature gaps, usage errors, and missing project directories — each with symptom, cause, and concrete recovery commands. Entries that have a stable CLI error code tag it in the heading (9 of 15) so CLI scripts can branch on `--json error_code` instead of grepping error prose; the other 6 (stale index, stale fold metadata, non-UTF-8 input, permission-denied scan skips, oversize-file scan errors) are documented by the specific symptom string or `status --json` field to watch because they are warnings, status fields, or `--verbose` scan diagnostics rather than coded CLI errors. MCP clients (which today receive `isError: true` text without a structured `error_code` field, and whose message text is not guaranteed to embed the bracketed CLI constant) match the documented MCP message recorded under each entry. The section links back to the canonical [Error codes](USER_GUIDE.md#error-codes) table under `## Options` rather than duplicating it.

## 日本語

- **USER_GUIDE.md にエラーコード分類と相互参照されたトラブルシューティング節を追加しました (#1579)** — `USER_GUIDE.md` に `## トラブルシューティング` 節（英日両方）を追加し、代表的な失敗モード 15 件（DB 未検出 / 古い / ロック / 書き込み不可 / 破損、ディスク満杯・temp-store 枯渇、スキーマが新しすぎる、fold メタデータが古い、FTS5 構文エラー、非 UTF-8 / 権限不足 / サイズ超過によるスキップ、トリムビルドの機能欠落、引数エラー、対象ディレクトリ欠落）を症状・原因・具体的な復旧コマンドの形で網羅しました。安定 CLI エラーコードを持つ 15 件中 9 件は見出しに `E001`〜`E011` を併記し、CLI スクリプトが `--json` の `error_code` で分岐できるようにしています。残り 6 件（古いインデックス、古い fold メタデータ、非 UTF-8 入力、権限不足によるスキャンスキップ、サイズ超過のスキャンエラー）はコードを持たない警告・ステータスフィールド・`--verbose` スキャン診断であるため、監視対象となる具体的な症状文字列や `status --json` フィールドを記載しています。MCP クライアントは現状 `isError: true` のテキストのみで構造化された `error_code` フィールドを受け取らず、メッセージ本文に CLI 側の角括弧付き定数が必ず含まれる保証もないため、MCP 側に構造化フィールドが追加されるまでは各項目に記録した MCP メッセージ本文と照合する運用としています。表の重複は避け、正準なリファレンスは `## オプション一覧` 内の [エラーコード](USER_GUIDE.md#エラーコード) へリンクしました。
