## English

- **USER_GUIDE.md adds a Troubleshooting section cross-referenced to the error-code taxonomy (#1579)** — `USER_GUIDE.md` now has a `## Troubleshooting` section (English + 日本語) cataloging 15 common failure modes — missing/stale/locked/non-writable/malformed databases, disk-full / temp-store exhaustion, schema-too-new, stale fold metadata, FTS5 syntax errors, non-UTF-8 / permission-denied / oversize skips, trimmed-build feature gaps, usage errors, and missing project directories — each with symptom, cause, and concrete recovery commands. Every entry tags its stable `E001`–`E011` code so CLI scripts can branch on `--json error_code` instead of grepping error prose, while MCP clients should match the bracketed constant in `isError: true` text until MCP exposes a structured field. The section links back to the canonical [Error codes](USER_GUIDE.md#error-codes) table under `## Options` rather than duplicating it.
