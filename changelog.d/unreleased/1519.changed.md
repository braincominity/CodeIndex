## English

- **`search` is now strict by default for every script; FTS5 prefix expansion is opt-in via `--prefix` or trailing `*` (#1519)** — The literal-safe `search` path no longer auto-upgrades CJK (or any other) tokens to an FTS5 prefix phrase. Searching `cdidx search 計算` against content containing `計算する` / `計算機` / `計算結果` now matches only the indexed token `計算` and stops widening into longer neighboring tokens. To opt into prefix expansion, append `*` to a single token (`cdidx search 計算*` promotes that one token) or pass the new `--prefix` flag / MCP `prefix` argument to widen every token in the query. `--prefix` cannot be combined with `--exact` / `--exact-substring` (those bypass FTS5 entirely) and is rejected with a usage-error hint. The previous unconditional CJK prefix promotion existed because the default FTS5 unicode61 tokenizer keeps a run of adjacent CJK codepoints as a single token, but that silently widened exact CJK identifier lookups and broke relevance for users hunting precise names; the new opt-in model preserves the broad-match capability while restoring strict semantics by default.

## 日本語

- **`search` の既定挙動を全スクリプトで strict 化し、FTS5 prefix 拡張は `--prefix` または末尾 `*` での opt-in に変更しました (#1519)** — literal-safe な `search` 経路は CJK（および他のスクリプト）のトークンを自動で FTS5 prefix phrase に昇格させなくなりました。`計算する` / `計算機` / `計算結果` を含むコードに対して `cdidx search 計算` を実行しても、indexed token `計算` だけにマッチし、より長い隣接トークンへの自動拡張は行いません。prefix 拡張をオプトインするには、トークン末尾に `*` を付ける（`cdidx search 計算*` でそのトークンだけ昇格）か、新しい `--prefix` フラグ / MCP の `prefix` 引数（クエリ全体の全トークンを昇格）を渡してください。`--prefix` は `--exact` / `--exact-substring` と併用できず（exact は FTS5 を経由しないため）、ヒント付きの usage error で拒否します。従来は FTS5 既定の unicode61 トークナイザが連続 CJK コードポイントを 1 トークンとして扱う制約から CJK トークンを無条件に prefix 昇格させていましたが、それにより CJK 識別子の厳密検索が静かに広がって精密な検索の relevance を損なっていました。今回の opt-in 化により広い検索能力を維持しつつ、既定では strict セマンティクスを取り戻します。
