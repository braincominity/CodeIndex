## English

- **`cdidx index --watch` keeps the index live by re-running incremental updates as files change (#1576)** — after the initial scan completes, the command stays running and uses `FileSystemWatcher` (FSEvents on macOS, inotify on Linux, ReadDirectoryChangesW on Windows) to detect creates, edits, renames, and deletes anywhere under the project root. Events are coalesced through a debounce window (`--debounce <ms>`, default 500 ms) so a burst of saves becomes one batched `--files` update; gitignore / `.cdidxignore` rules and SkipDirs are honored, the per-DB `IndexLock` is released between batches so other cdidx commands can still query, and watcher-buffer overflow falls back to a full incremental rescan rather than silently dropping changes. The loop emits `status: "watching" / "updated" / "rescanned" / "overflow" / "stopped"` JSON events when `--json` is set (otherwise a human-readable `[watch] …` line on stderr per batch) and exits cleanly on Ctrl+C / SIGTERM. The flag rejects `--commits`, `--files`, and `--dry-run` with an actionable hint because watch already drives continuous incremental updates.

## 日本語

- **`cdidx index --watch` でインデックスをライブ更新できるようになりました (#1576)** — 初回スキャン完了後もコマンドが実行を継続し、`FileSystemWatcher`（macOS は FSEvents、Linux は inotify、Windows は ReadDirectoryChangesW）でプロジェクト配下のファイル作成・編集・リネーム・削除を検知します。イベントはデバウンス窓（`--debounce <ms>`、既定 500 ms）でまとめられ、一連の保存がひとつの `--files` 更新にバッチ化されます。gitignore / `.cdidxignore` と SkipDirs を尊重し、バッチ間で DB ごとの `IndexLock` を解放するため他の cdidx コマンドからの問い合わせも妨げません。Watcher バッファのオーバーフロー時は変更を黙って捨てる代わりにフル増分再走査へフォールバックします。ループは `--json` 時に `status: "watching" / "updated" / "rescanned" / "overflow" / "stopped"` の JSON を出力し（そうでなければ stderr にバッチごとの `[watch] …` 行を出力）、Ctrl+C / SIGTERM で正常終了します。watch は連続的な増分更新を内蔵しているため `--commits` / `--files` / `--dry-run` との併用は実行可能な代替案を添えて拒否します。
